# backend/

FastAPI application serving normalized, pre-aggregated incident data from PostgreSQL. Read-only API -- no user accounts, no writes from the public (see `../docs/ARCHITECTURE.md`).

Note: this is plain PostgreSQL, not PostGIS. All spatial computation (the point-in-polygon join) already happens offline in the Milestone 2 pipeline (`../scripts/geospatial_join.py`) -- the database only stores and returns precomputed neighborhood boundaries as GeoJSON (a plain JSON column), so it never needs a live geospatial query engine. See `../docs/ARCHITECTURE.md` for the reasoning (this also sidesteps a genuinely painful PostGIS/GDAL from-source build on unsupported platforms).

## Local setup

This machine doesn't support Docker Desktop (macOS 13, below Docker's minimum of Sonoma), so local development uses a native Postgres install instead of the `docker-compose.yml` at the repo root (that file is still valid for anyone with Docker -- point `DATABASE_URL` at it instead).

### 1. Get a local Postgres running (one-time)

If you already have a Postgres server running locally (Postgres.app, Homebrew, Docker, etc.), you just need a database:

```bash
createdb repeat_signal
```

Otherwise, install one via Homebrew:

```bash
brew install postgresql@16
brew services start postgresql@16
createdb repeat_signal
```

No PostGIS extension needed -- see the note above.

### 2. Set up the Python environment

```bash
cd backend
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # adjust DATABASE_URL if your local Postgres setup differs
```

### 3. Run migrations and seed the database

Requires the data pipeline (`../scripts/run_pipeline.py`) to have been run at least once, since seeding reads `../data/processed/neighborhoods.parquet` and `monthly_aggregates.parquet`.

```bash
alembic upgrade head
python seed.py
```

### 4. Run the API

```bash
uvicorn app.main:app --reload
```

Visit `http://localhost:8000/docs` for interactive API documentation (auto-generated by FastAPI), or `http://localhost:8000/api/health` for a basic health check.

### 5. Run tests

Tests run against the same local dev database (not an isolated test database -- a deliberate scope simplification for this project, see `tests/conftest.py`). Make sure it's migrated and seeded first, then:

```bash
pytest
```

## Endpoints

| Endpoint | Description |
|---|---|
| `GET /api/health` | Basic health check |
| `GET /api/cities` | Per-city totals and date ranges |
| `GET /api/categories` | Shared taxonomy categories with totals (see `../scripts/category_mapping.py`) |
| `GET /api/neighborhoods?city=` | Neighborhood listing, optionally filtered by city |
| `GET /api/neighborhoods/{id}` | Single neighborhood detail, including boundary geometry as GeoJSON |
| `GET /api/neighborhoods/geojson?city=&category=` | Neighborhood boundaries + incident totals as one GeoJSON `FeatureCollection`, geometry simplified for web delivery -- powers the Milestone 6 map. Fetched per city on demand, not all 380 neighborhoods at once. |
| `GET /api/aggregates?city=&category=&neighborhood_id=&start_date=&end_date=` | Filtered monthly incident counts -- the main data endpoint |
| `GET /api/summary` | Overall totals across all three cities |
| `GET /api/trends/monthly?city=&category=` | Total incidents per city per month, summed across neighborhoods -- powers the Milestone 7 time-series chart. Excludes the current (necessarily incomplete) calendar month. |
| `GET /api/trends/categories?city=` | Total incidents per city per category, summed across neighborhoods and months -- powers the Milestone 7 category comparison chart. |

## Schema migrations

Uses Alembic. To add a new migration after changing `app/models.py`:

```bash
alembic revision --autogenerate -m "describe the change"
alembic upgrade head
```
